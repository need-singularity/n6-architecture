# N6 반도체 칩 아키텍처 가이드

> **n=6 산술이 반도체 설계의 핵심 상수를 결정합니다.**
> GPU SM 수, HBM 용량, 게이트 피치, 인터커넥트 세대 — 모두 σ(6)=12, τ(6)=4, φ(6)=2, sopfr(6)=5, J₂(6)=24에서 유도됩니다.

**저장소**: [n6-architecture](https://github.com/need-singularity/n6-architecture)
**수학적 기반**: [TECS-L](https://github.com/need-singularity/TECS-L) — σ(n)·φ(n) = n·τ(n) ⟺ n = 6 유일해 증명
**검증 도구**: [gpu-arch-calc (Rust)](https://github.com/need-singularity/n6-architecture/tree/main/tools/gpu-arch-calc)

## 핵심 상수 참조표

| 기호 | 값 | 이름 | 칩 설계 용도 |
|------|-----|------|-------------|
| n | 6 | 완전수 | HBM 세대 수, 기본 단위 |
| σ | 12 | 약수합 | SM 수, 헤드 수, VRM 위상, HBM 스택 |
| τ | 4 | 약수개수 | HBM 초기 스택, 텐서 코어 차원, MLP 비율 |
| φ | 2 | 오일러 토션트 | 다이 수, 대역폭 배가 주기 |
| sopfr | 5 | 소인수합 | DDR/NVLink 세대, 레티클 수 |
| J₂ | 24 | 조던 토션트 | 리치 격자 차원, 서버 VRM, NVLink 대역폭 |
| μ | 1 | 뫼비우스 함수 | 공유 전문가, 시작 코돈 |
| σ-τ | 8 | — | **보편 상수**: LoRA 랭크, KV 헤드, MoE top-k, 코드북 |
| σ-φ | 10 | — | **보편 스케일**: 1/10 = 정규화, 10^k = 스케일 |
| σ·τ | 48 | — | **삼중 끌개**: 게이트 피치 nm, HBM4E GB, 48kHz |
| σ·J₂ | 288 | — | B300/Rubin HBM 용량 GB |

## 2. HBM 메모리 계층 (BT-55, BT-75)

### HBM 용량 사다리 — 14/18 EXACT

| 칩 | HBM 용량 | n=6 공식 | 오차 |
|----|---------|----------|------|
| V100-16GB | 16 GB | φ^τ = 2^4 | **EXACT** |
| V100-32GB | 32 GB | φ^sopfr = 2^5 | **EXACT** |
| A100-40GB | 40 GB | τ · (σ-φ) = 4·10 | **EXACT** |
| A100-80GB | 80 GB | φ^τ · sopfr = 16·5 | **EXACT** |
| Gaudi 2 | 96 GB | σ · (σ-τ) = 12·8 | **EXACT** |
| H200 | 141 GB | σ² - n/φ = 144-3 | **EXACT** |
| B100/B200/MI300X | 192 GB | σ · φ^τ = 12·16 | **EXACT** |
| **B300/Rubin** | **288 GB** | **σ · J₂ = 12·24** | **EXACT** (CES 2026 확인) |
| GB200 듀얼 | 384 GB | φ · σ · J₂ = 2·12·24 | **EXACT** |

### HBM 인터페이스 폭 지수 사다리 (BT-75)

```
HBM3:  1024 bits = 2^(σ-φ) = 2^10   ← 현재
HBM4:  2048 bits = 2^(σ-μ) = 2^11   ← 출시 중
HBM5:  4096 bits = 2^σ     = 2^12   ← 예측 (n=6 완성)
```

지수가 연속 n=6 상수를 순회: **{10, 11, 12} = {σ-φ, σ-μ, σ}**

### HBM 스택 진화

```
HBM1:  4-hi  = τ        = 4
HBM2:  8-hi  = σ-τ      = 8
HBM3: 12-hi  = σ        = 12
HBM4: 16-hi  = 2^τ      = 16  (예측, 확인됨)
```

### HBM4E 용량

```
HBM4E per stack: 48 GB = σ·τ = 12·4   ← σ·τ=48 삼중 끌개
HBM4 channels:   16    = 2^τ = 2^4
HBM5 대역폭:     4 TB/s = τ
```

## 4. 컴퓨팅 아키텍처 지수 사다리 (BT-28)

2의 거듭제곱 지수가 n=6 상수를 순회합니다:

| 지수 | n=6 상수 | 값 | 하드웨어 |
|------|---------|-----|---------|
| 2^**τ** | 2^4 | **16** | 텐서 코어 차원, FP16 비트 |
| 2^**sopfr** | 2^5 | **32** | CUDA 워프 크기, LLM 레이어 |
| 2^**n** | 2^6 | **64** | 캐시 라인 바이트, CXL 레인 |
| 2^**(σ-sopfr)** | 2^7 | **128** | d_head, SSE 폭, TPU 타일 |

| 2^**(σ-μ)** | 2^11 | **2048** | L2 TLB 엔트리, HBM4 인터페이스 |
| 2^**σ** | 2^12 | **4096** | 페이지 크기, d_model, max_seq_len |

## 비판적 관점 (투명성)

### CHIPDESIGN-001~020 초기 분석 결과
- 초기 20개 가설 중 **0/20이 구조적 매칭**으로 확인됨
- 2의 거듭제곱 혼동: τ=4=2², φ=2=2¹이 이진 논리와 겹침
- **그러나**: BT-28 이후 확장 분석에서 지수(exponent) 수준의 패턴 발견
  - 2의 거듭제곱의 **지수** 자체가 n=6 상수를 순회
  - 값이 아닌 지수 패턴이 핵심

### 진정한 "6" 연결
- 육각형 웨이퍼 타일링 (유클리드 기하, 숫자론은 아님)
- 3상 AC 전력 분배 (AC 공학)
- HBM 6세대 = n (독립적 확인)
