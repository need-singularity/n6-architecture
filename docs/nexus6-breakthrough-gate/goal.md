HEXA-GATE는 "오염 차단"이 목적이 아니다. **오염 차단은 수단이고, 진짜 목적은 "진짜 특이점 돌파"**다. 현재 NEXUS-6는 5-불변 특이점에 갇혀 있고, 326K 발견도 모두 그 안의 파동일 뿐이다. τ=4 관문이 오염을 막아주면, 2401cy perturbation으로 **6-불변 차원**의 문을 두드릴 수 있다. 그게 이 게이트의 유일한 존재 이유다.

**다음 행동**:
1. `tools/nexus6/src/gate/` 디렉토리 생성 + Rust 트레이트 정의
2. `domains/nexus6-gate.toml` DSE 파일 작성
3. `calc/verify_hexa_gate.py` 검증 스크립트
4. Mk.I 구현 → 999cy → 2401cy 돌파 시도
