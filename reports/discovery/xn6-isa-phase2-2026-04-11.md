## 3. 24 명령어 카테고리 요약

### 3.1 funct3 × variant 매트릭스 (6 × 4 = 24 = σ·φ = n·τ)

```
funct3  원시         var=00      var=01      var=10      var=11
000     Ω₁ GEMM      gemm.m      gemm.a      gemm.tc     gemm.bf16
001     Ω₂ SOFTMAX   sm.exp      sm.max      sm.sub      sm.norm
010     Ω₃ TOPK      tk.sort     tk.pick     tk.merge    tk.idx
011     Ω₄ GATE      gt.mul      gt.and      gt.or       gt.xor
100     Ω₅ REDUCE    rd.sum      rd.max      rd.min      rd.mean
101     Ω₆ CONV6     conv.3x3    conv.1x1    conv.dw     conv.pw
110     RSVD         ─           ─           ─           ─
111     RSVD         ─           ─           ─           ─
```

### 3.2 카테고리별 의미

| funct3 | 원시 | 카테고리 요지 | 평균 사이클 |
|--------|------|--------------|------------|
| 000 | GEMM | 12×12 master/accumulate/async TC/BF16 4 변종 | σ=12 |
| 001 | SOFTMAX | Phi6 exp → max → sub → norm 체인 4 단계 | φ=2 |
| 010 | TOPK | 정렬·top-φ·병합·인덱스 추출 (MoE router) | φ²=4 |
| 011 | GATE | mul·AND·OR·XOR 희소 마스크 4 변종 | μ=1 |
| 100 | REDUCE | sum·max·min·mean τ=4 reduce | τ=4 |
| 101 | CONV6 | 3x3·1x1·depth-wise·point-wise 4 kernel | n=6 |

전체 평균 CPI ≈ 29/6 ≈ φ² = 4, 목표 P-ISA-1 (CPI ≤ φ=2) 은 Phase 3 파이프라인 병렬화로 달성 예정.

### 3.3 n=6 수식 정합성 24 건

- **Ω₁ GEMM** 4 종: σ² = n·τ·φ² = 144 MAC, BF16 지수 σ-τ=8
- **Ω₂ SOFTMAX** 4 종: Phi6 (x²-x+1) φ=2 사이클, Egyptian 역수 LUT
- **Ω₃ TOPK** 4 종: top-φ=2 MoE, tier-A n=6 슬롯
- **Ω₄ GATE** 4 종: Boltzmann 1/e, Möbius μ(6)=1, Mertens, radical
- **Ω₅ REDUCE** 4 종: τ=4 reduce tree depth=φ=2, mean = shift-τ-bit
- **Ω₆ CONV6** 4 종: n=6 탭, τ=4 채널, σ=12 길이, σ·τ=48 MAC depth-wise

## 4. 비트 인코딩 요약

RISC-V R-Type 32 비트 / opcode = 0x0B (custom-0) 고정:

```
 31       25 24   20 19   15 14  12 11    7 6      0
┌──────────┬───────┬───────┬──────┬───────┬────────┐
│  funct7  │  rs2  │  rs1  │funct3│  rd   │ opcode │
│  7 bit   │ 5 bit │ 5 bit │3 bit │ 5 bit │ 7 bit  │
└──────────┴───────┴───────┴──────┴───────┴────────┘

funct7 = [reserved:4 | n6:1 | variant:2]
       = 0b0000_1vv  (v=00~11 = τ 선택)
기본 funct7 = 0x04, 0x05, 0x06, 0x07

funct3 ∈ {000,001,010,011,100,101} → 6 = n 개 원시연산 사용
funct3 ∈ {110,111}                 → 2 개 예약 (BT-380 메타 승격)
```

- 전체 custom-0 공간 = 24 (현재) + funct7 상위 4 비트 × funct3 2 예약 = 384 = σ² + σ·τ·φ 확장 여지.
- `rsvd` 필드 (funct7[6:3]=0000) 는 Phase 4 Xn6-MEM/DMA 용.

## 5. GCC intrinsic 샘플 3 개

### 샘플 1 — Phi6 softmax 체인 (Ω₂ 4 명령 fuse)

```c
#include <xn6intrin.h>

// 12 원소 벡터에 Phi6 기반 softmax 적용
// 전통 softmax 14 사이클 → Xn6 4 명령 × 평균 φ=2 = 8 사이클 (1.75배)
// + Phi6 근사로 GELU 7배 절감 → 총 실효 ~14배
void softmax_phi6_vec12(float* dst, const float* src) {
    float tmp[12];
    float max_val;

    __builtin_xn6_sm_exp (tmp, src);              // sm.exp   (φ=2)
    __builtin_xn6_sm_max (&max_val, tmp);         // sm.max   (φ²=4)
    __builtin_xn6_sm_sub (tmp, tmp);              // sm.sub   (1)
    __builtin_xn6_sm_norm(dst, tmp);              // sm.norm  (φ=2)
}
```

### 샘플 2 — 12×12 BF16 GEMM (Ω₁ training 주력)

```c
#include <xn6intrin.h>

// LLM training 의 핵심 — BF16 precision 12×12 행렬곱 1 타일
// σ=12 사이클에 σ²=144 MAC 완료, 기존 256 스칼라 곱 대비 ~12배
void bf16_gemm_tile(__bf16* c, const __bf16* a, const __bf16* b) {
    // 1 명령 = 1 타일 = 144 MAC (σ²)
    __builtin_xn6_gemm_bf16(c, a, b);
}

// 대형 행렬 분해 호출: N×N 타일링
void bf16_matmul_large(__bf16* C, const __bf16* A, const __bf16* B, int N) {
    for (int i = 0; i < N; i += 12) {
        for (int j = 0; j < N; j += 12) {
            __builtin_xn6_gemm_m(&C[i*N+j], &A[i*N], &B[j]);        // 첫 타일
            for (int k = 12; k < N; k += 12) {
                __builtin_xn6_gemm_a(&C[i*N+j], &A[i*N+k], &B[k*N+j]);  // 누적
            }
        }
    }
}
```

### 샘플 3 — MoE router (Ω₃ tk.pick + Ω₄ gt.mul fuse)

```c
#include <xn6intrin.h>

// Egyptian MoE router — σ=12 전문가 스코어에서 top-φ=2 선택 후
// 게이팅 마스크 적용. 전통 softmax+topk 14+10 → Xn6 2+1 = 3 사이클
void egyptian_moe_dispatch(float* token_out,
                           const float* token_in,
                           const float* expert_scores,  // σ=12
                           float* expert_cache) {
    int top2_idx[2];
    __builtin_xn6_tk_pick(top2_idx, expert_scores);   // tk.pick (φ=2)

    // 전문가 2 개만 활성 (Boltzmann gate)
    float mask[12] = {0};
    mask[top2_idx[0]] = 1.0f;
    mask[top2_idx[1]] = 1.0f;

    __builtin_xn6_gt_mul(token_out, expert_cache, mask);  // gt.mul (μ=1)
}
```

## 6. hexa-lang `@xn6` attr 스타일 (R12 AI-NATIVE)

```
@xn6(op="gemm_m")
fn matmul12(a: tensor(12,12), b: tensor(12,12)) -> tensor(12,12) {
    return gemm(a, b)
}
```

컴파일러는 `@xn6` attr 를 보면 자동으로 대응 `__builtin_xn6_*` intrinsic 삽입 → 사용자는 **수동 asm 0 줄** 으로 24 명령 전부 사용 가능. R12 준수.

## 7. Phase 1 과의 매핑 차이

| 축 | Phase 1 섹션 6.2 | Phase 2 (본 세션) |
|----|------------------|------------------|
| 변종 명명 | 정밀도/알고리즘 (gemm.f16, smax.phi6, ...) | 물리 동작 (gemm.m/a/tc/bf16, sm.exp/max/sub/norm) |
| 매트릭스 구조 | funct3×variant=24 | **동일** (불변) |
| 핵심 수식 | σ·φ=n·τ=24 | **동일** (불변) |
| opcode 공간 | custom-0 (0x0B) | **동일** (불변) |
| funct7 구조 | [variant:2 | n6:1 | rsvd:4] | **동일** |
| Phase 1 md 수정 | — | **수정하지 않음** (Phase 2 는 신규 파일만) |

> Phase 2 mnemonic 이 Phase 1 section 6.2 와 다른 이유: Phase 1 은 "어떤 정밀도" 기준 초안, Phase 2 는 "어떤 물리 동작" 기준 구현. Mk.II 디코더부터는 Phase 2 SSOT 사용 (`xn6_isa_24_spec.md`). Phase 1 md 는 설계 히스토리 아카이브.
